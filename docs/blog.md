# 我用 6 周做了一个会自我评审的 AI Agent

> 0 前端基础，全靠 vibe coding。这是一份给同样想入门 AI 应用开发的「非程序员」的实战记录。

## 一、为什么是写信？

ChatGPT 横空出世两年了，市面上的 AI 写作工具基本是同一个套路：

> 「请输入主题，AI 帮你生成 500 字。」

我试过用 ChatGPT 给妈妈写一封信，结果它写出来的东西**任何人都能用** —— "亲爱的妈妈，您辛苦了，您是世界上最伟大的母亲" —— 这种话和打印机吐出来的没区别。

问题不在模型，**问题在交互**。一个真正能写出动人书信的助手，不应该是「告诉我主题 → 给你范文」，而应该是「**像朋友一样问你具体发生了什么**」。

这就是这个项目的起点。

## 二、6 周里走过的 4 个阶段

### Week 1-2：能用 vs 好用

最开始是最简单的形态：表单 → 提交 → 等返回完整文本。

写完那一刻我就发现两个问题：

1. **等待感太强**：调一次 DeepSeek 接口，要等 20 秒才有反应，用户当场就走了
2. **输入门槛太高**：表单有 6 个字段，让用户一次填完，他根本不会写"具体事件"那栏

解决方案：
1. **流式输出**：用 `ReadableStream` 让模型边想边吐，用户看到字一个个浮现，等待感消失大半
2. **改对话**：把表单拆成 4-5 轮对话，每次只问一个问题

第 2 点是关键。**让用户填表单，是工程师的偷懒；让用户聊天，才是产品经理的功夫。**

### Week 3：第一次接触 "Agent" 这个概念

> "Agent" 这个词被滥用得很严重。最初我以为 Agent 就是"能多轮聊天的 GPT"，做完 Week 3 才知道完全不是。

Week 3 我做了一件事：让一个 LLM 扮演「引导员」收集信息，**收集完后输出一个特殊信号** `[[READY]]` 加 JSON 素材；后端检测到信号，**自动切换成「写作者」Prompt**，开始写信。

```typescript
// 后端伪代码
const listenerOutput = await llm.call(listenerPrompt + history);
if (listenerOutput.match(/\[\[READY\]\]\s*(\{[\s\S]+\})/)) {
  const collected = JSON.parse(matched[1]);
  // 切换角色！
  return await llm.stream(writerPrompt + collected);
}
```

这段代码让我第一次理解了什么叫「**让 LLM 主导流程，但代码控制节奏**」。

但严格说，**这还不算 Agent**。这只是「两个 Prompt 用一段 if 串起来」。

---

### Week 4：Self-Critique Loop — 这才是 Agent 的味道

Week 3 的引导员 + 写作者已经能写出像样的信，但还有一个老问题：**模型偶尔会写出套路文**，比如"亲爱的妈妈，你为我做了那么多……"

我能凭直觉判断"这封信写糟了"，但**怎么让程序也能判断**？

答案是：**再加一个 LLM 角色作为评审师，让它输出 JSON 评分**。

```typescript
// Polisher Prompt 的核心
"你是严苛的中文书信编辑，从不放水。请按 4 个维度（每项 0-25）评判：
1. 具体细节  2. 反套路  3. 情感真实  4. 完成度
严格输出 JSON：{score: number, issues: string[], suggestions: string}"
```

然后在代码里：

```typescript
if (polishResult.score < 80) {
  await streamRewriter(prevDraft, polishResult); // 带反馈重写
}
```

这一刻才是真正的 "Agent" 雏形 —— **模型评判模型的输出，并基于评判改变下一步行为**。

业内把这个模式叫 **Self-Critique** 或 **Reflexion**（Shinn 等人 2023 年的论文）。

### Week 5：风格 —— 真正的产品差异化

到这里产品技术架构已经稳了，缺一个"杀手锏"。

灵感来自我女朋友的一句话：

> "你这个东西不错，但写出来都一个味道。"

她说得对。再厉害的 Agent，写出来都是同一种 LLM 腔调，没有作家个人风格。

解决方案是 **Prompt 注入风格**：

```typescript
const STYLES = {
  zhuziqing: `请模仿朱自清《背影》《给亡妇》的笔调：
- 用具体的日常细节代替抒情
- 句子短，节奏慢
- 情感深而不外露，常在最朴素的描述里突然停一下...`,
  
  wangxiaobo: `请模仿王小波给李银河的情书笔调：
- 既深情又自嘲，用幽默化解郑重
- 时不时蹦出一个看似不正经、实则击中要害的比喻
- 真挚但绝不煽情，绝不用"我爱你"这种直白话...`,
  // ...
};
```

实测，同一段素材（妈妈凌晨 5 点包饺子），王小波风格写出来的是：

> "可我全吃了，就着凉水。那皮儿有点硬了，但嚼着嚼着，我他妈差点没绷住。"

朱自清风格写出来的是：

> "我从来不敢算，这些年你塞给我多少东西。"

**两个 Prompt 片段，给同一个模型注入了两个作家的灵魂。**这就是 LLM 时代产品的杀手锏 —— 不是模型变了，是 Prompt 变了。

## 三、6 周后我学到的 3 件最重要的事

### 1. Agent ≠ 多轮对话

我以前以为 Agent 就是"能记上下文的 GPT"。错了。

**Agent 的定义是：能基于 LLM 输出来改变自己下一步行为的程序。**

最简单的 Agent 是 if/else（我的 W4 Polisher Loop）。
中等的 Agent 是 Tool Use（让模型选择调用什么外部函数）。
最高级的 Agent 是 ReAct（让模型自主规划多步行动）。

只是聊天没记忆都不算 Agent，那只是 Chat。

### 2. 工程的复杂度都在「边界」上

LLM 调用本身只有 5 行代码。复杂的是：
- **流式协议**：边返回文本边切阶段，需要自定义标记
- **JSON 提取的鲁棒性**：模型偶尔会在 JSON 前后加废话
- **失败重试**：模型可能输出格式错误，整个 Loop 卡住
- **Token 控制**：Polisher 不需要全量历史，否则成本指数爆炸

业内一句话：**做 AI 应用，80% 的时间是处理 LLM 的"不靠谱"**。

### 3. Prompt 工程是真功夫

刚开始我以为 Prompt 就是写两句话给模型。错了。

一个好的 Writer Prompt 需要：
- **明确角色**："你是一位中文书信写作大师"
- **结构化输入**：把素材列成 bullet
- **负面约束**："不要套路化开头，不要此致敬礼"（**反过来教比正向教更有效**）
- **正面引导**："多用具体细节代替抽象情感"
- **格式锁死**："300-500 字，第一人称中文"

一段 200 字的 Prompt 决定了一个产品的天花板。**写 Prompt 这件事，未来 3 年都不会被 AI 自己取代。**

## 四、下一步：从 Workflow 升级到真正的 Agent

老实说，我目前做的还是 **Agentic Workflow（编排式 Agent）** —— 路径是我写死在代码里的。真正的 **Autonomous Agent** 是让 LLM 自己决定下一步用什么工具。

下一阶段计划：

1. **Tool Use**：给模型几个工具（搜古诗 / 查历史关系 / 获取当前日期），让模型自己决定要不要用
2. **Long-term Memory**：用户写过的信会被提取成"画像"，下次写信自动调用
3. **多模型协作**：Polisher 用 GPT-4，Writer 用 DeepSeek，省钱又准

但这是下个 6 周的事。

---

## 五、给同样想入门的你

如果你也是 0 基础但想做 AI 应用，我给 4 个建议：

1. **先做能用的，再做好用的**。我 Week 1 的版本丑得不能看，但它能跑通，这给了我巨大的正反馈
2. **每周一个明确的「能给别人演示的功能」**。不要憋大招，憋出来都没了
3. **写代码用 AI 但不要让 AI 代写所有代码**。看不懂的部分必须当场问清楚，否则技术债 W3 就开始反噬
4. **写文档比写代码重要**。这篇博客是我整个项目的 ROI 最高的部分 —— 它是我以后找工作的真正凭证
